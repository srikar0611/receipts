A lightweight evidence trail

From agent session to review evidence in three steps.

Receipts stays out of the agent’s way. It observes a terminal session, records the facts it can support, and gives the reviewer a compact artifact to inspect.

01

Capture the session

Wrap Codex, Claude Code, or another coding command in a POSIX PTY. Commands, transcript, and Git snapshots are captured locally.

$ receipts run -- codex "fix login"
02

Find the gap

Deterministic analysis maps convention-named tests to changed source files and asks whether a test ran after the last edit.

→ manifest + sha256 receipt
03

Review the proof

Render a Trust Card, open an offline replay, or post a sticky PR comment with the included GitHub Action.

→ card · replay · PR comment

Replay the evidence

A session is easier to trust when you can inspect its timeline.

The replay is one static HTML file with the recorded manifest embedded. It makes the sequence of edits, test runs, and flags legible without standing up a backend.
